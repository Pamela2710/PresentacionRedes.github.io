¿Qué es?

Una red mallada es una red WiFi compuesta por varios componentes, pero todos utilizando el mismo SSID o nombre de la red, y la misma contraseña. Por una parte tienes el router o estación base que recibe la señal del módem o router de tu operadora, y luego tienes los nodos, también llamados satélites o puntos de acceso, que son los que hacen de repetidores. La idea es que entre todos crean una red de tipo malla. Esto quiere decir que no se limitan a que el router le envíe la señal a los nodos y estos la difundan. No, los nodos también se van a poder comunicar entre sí además de con el router para optimizar la cobertura lo máximo posible y optimizar las conexiones.



Bibliotecas utilizadas

  • py2p: En esta red se busca que cada nodo se comunique directamente con otros nodos cercanos en lugar de tener que entablar comunicacion a traves de un servidor central.
  • threading: Es un módulo en Python que proporciona clases y funciones para trabajar con hilos (también conocidos como threads).
  • time: Permite importar el módulo de tiempo estándar de Python. Este módulo proporciona funciones para trabajar con el tiempo, como medir la duración de una operación o hacer pausas en la ejecución de un programa.
  • uuid: Es un módulo en Python que proporciona funciones para trabajar con identificadores únicos universales (UUID).


Métodos


handle_message(msg, sender)

Esta función recibe un mensaje (msg) y su remitente (sender). Su objetivo es procesar y mostrar el contenido del mensaje recibido. Primero, extrae el contenido del mensaje y el remitente limpiando los caracteres no deseados y luego muestra el remitente y el mensaje en la consola.

sender_to_string(message_obj)

Esta función recibe un objeto message_obj y devuelve una representación en forma de cadena de los primeros 8 caracteres de ese objeto. El propósito exacto de esta función no está claro en el código proporcionado.

show_connected_devices(node)

Esta función recibe un objeto node que representa un socket de la malla y muestra una lista de los dispositivos conectados actualmente en la malla. Recorre la tabla de enrutamiento del nodo e imprime el ID de cada dispositivo conectado en la consola.

message_to_string(message_obj)

Esta función recibe un objeto message_obj y devuelve una representación en forma de cadena de los paquetes del mensaje. Al igual que sender_to_string(), el propósito exacto de esta función no está claro en el código proporcionado.

message_handler()

Se ejecuta en un hilo y está constantemente esperando mensajes entrantes y los pasa a handle_message() para su procesamiento.

reconnection_handler()

Se ejecuta en un hilo y se encarga de intentar reconectar con el nodo de arranque si no hay conexiones en la tabla de enrutamiento.




Ejecución

Instrucciones para ejecutar el programa
  • Definir el nodo de arranque en un cliente.
  • En el segundo cliente, al ingresar la direccion ip y puerto del nodo de arranque.
  • Del tercer cliente en adelante, ingresar la direccion ip y puerto del nodo anterior

Al iniciar el programa, crea un nodo mesh en el puerto 5678. Muestra el ID del nodo, posteriormente se solicita ingresar la dirección (IP:Puerto) del nodo bootstrap. Este es el nodo que va a servir como nodo de arranque. Si se deja en blanco el programa asume que la computadora es el nodo bootstrap y recupera la direccion ip automaticamente. Caso contrario el programa intentará conectarse con el nodo bootstrap.
